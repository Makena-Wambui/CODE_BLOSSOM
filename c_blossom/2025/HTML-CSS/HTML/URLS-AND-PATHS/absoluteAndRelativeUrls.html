An absolute URL points to a location defined by its absolute location on the Web, including protocol and a domain name.

Say you have a file, index.html that is uploaded to a directory called projects that sits at the root of a web server,
and the website's domain is https://www.example.com.
The page would be available at https://www.example.com/projects/index.html.
Or just https://www.example.com/projects since most web servers are configured to serve the index.html file by default.
An absolute URL will always point to the same location no matter where it is used.

Relative URLS point to a location that is relative to the file you are linking from.
Relative URLS point to different locations depending on the actual location of the file they are used in.
